# 诊断 → 可执行的高抛低吸行动计划（设计）

日期：2026-06-22
状态：已批准，待实现

## 背景与目标

当前「诊断」标签页（`/api/regime`）只输出行情形态标签（上涨/下跌/震荡）、区间收益、年化波动、均线关系，以及一句话策略建议。它**不给具体操作**。

本设计把诊断升级为一份**当下可执行的高抛低吸行动计划**：基于历史净值算出买/卖的净值阈值带，给出「现在该买/卖/持有 + 触发价位 + 买/抛多少」的前瞻指引，并结合行情形态做修正。

不预测未来走势，不构成投资建议（保留现有免责声明）。

## 三个核心决策（已确认）

1. **输出形态** = 当下行动计划（前瞻），而非历史操作点回放。
2. **阈值方法** = 均线 ±k·σ 波动带（布林带思路）。
3. **仓位规则** = 分档加码/减仓。

## 详细设计

### ① 阈值带（数学核心）

- 取近 `band_window`（默认 60 交易日）净值。
- 中轴 `MA` = 该窗口净值均值。
- 波动 `σ` = 该窗口净值（acc_nav）标准差（样本标准差，与 `analyze::stdev` 一致）。
- 四条触发线，围绕中轴对称：

| 线 | 净值位置 |
|---|---|
| 强力低吸线 | MA − 2σ |
| 低吸线 | MA − 1σ |
| 观望带 | (MA−1σ, MA+1σ) |
| 高抛线 | MA + 1σ |
| 强力高抛线 | MA + 2σ |

每条线都换算成具体净值数字，直接告诉用户「跌到 X 买、涨到 Y 卖」。

档位：内档 1σ、外档 2σ（固定，首版不进表单）。

### ② 分档仓位规则（买多少 / 抛多少）

- 买入基准 `base_amount`（默认 1000 元）；卖出基准 `sell_pct`（默认持仓 20%）。
- 触发 −1σ → 买 1× 基准（1000 元）；跌破 −2σ → 买 2× 基准（2000 元）。
- 触发 +1σ → 卖 1× 基准比例（20%）；突破 +2σ → 卖 2× 基准比例（40%）。

### ③「当下」判读（前瞻指引）

用最新净值算 `z = (nav − MA) / σ`，落在哪一档给出动作：

- `z ≤ −2` → 强力低吸：买 2× 基准
- `−2 < z ≤ −1` → 低吸：买 1× 基准
- `−1 < z < +1` → 观望：持有不动
- `+1 ≤ z < +2` → 高抛：卖 1× 基准比例
- `z ≥ +2` → 强力高抛：卖 2× 基准比例

动作卡包含：当前净值与日期、当前 z、当前动作与金额/比例、距下一档触发线还需涨/跌多少（百分比 + 目标净值）。

观望带时展示「距低吸线还差 −X%（到净值 a）/ 距高抛线还差 +Y%（到净值 b）」。

### ④ 结合历史 + 形态修正

- **结合历史**：统计该带在过去 `window` 天内触发低吸 N 次、高抛 M 次（净值穿越 −1σ / +1σ 线计数），佐证带子是「活的」。
- **形态修正**（复用现有 `detect_regime`）：
  - 震荡 → 全套高抛低吸（最适用）。
  - 上涨趋势 → 低吸照做；高抛减半执行（仅在 +2σ 才减仓），提示「顺势勿过早下车」。
  - 下跌趋势 → 高抛照做；低吸只在 −2σ 小额试探，显著风险提示。
- 修正以 `caveat`（提示文案）+ 对卖出/买入力度的标注呈现。
- 保留现有「不构成投资建议」免责声明。

## 测试（TDD）

- `analyze.rs` 单测：
  - 深跌净值序列 → 当前动作为「强力低吸」、金额 2× 基准。
  - 高位净值序列 → 当前动作为「高抛」。
  - 震荡序列 → 历史触发计数（buy_hits/sell_hits）> 0。
  - 形态修正：上涨趋势下卖出力度被削弱（caveat / 标注不同于震荡）。
  - 数据不足继续走现有「数据不足」错误路径。
- `web` 测试：`/api/regime` 返回 JSON 含 `plan.tiers`、`plan.current`。

## 默认值

`band_window=60`、档位 `1σ/2σ`、`base_amount=1000`、`sell_pct=0.20`。

## 不做（YAGNI）

- 不做历史操作点逐笔回放（用户选了前瞻形态）。
- 档位 σ、网格、固定金额卖出等不进首版表单。
